# CHEKA GIFU サポート管理アプリ 公開手順書

スマホで使えるようにするまでの手順です。
一度やれば次回からは自動更新されます。

---

## 📋 全体の流れ（約30分）

1. GitHubにコードを置く（5分）
2. Cloudflareに登録する（5分）
3. データベースを作る（5分）
4. アプリを公開する（10分）
5. 動作確認（5分）

---

## STEP 1：GitHubにコードを置く

### 1-1. GitHubアカウント作成（すでにある人はスキップ）
1. https://github.com を開く
2. 「Sign up」でアカウント作成
3. メールアドレス確認まで完了させる

### 1-2. 新しいリポジトリを作る
1. GitHubにログイン
2. 右上の「＋」→「New repository」をクリック
3. 以下のように設定：
   - Repository name: `fasting-app`
   - **Private にする**（重要！）
   - 「Create repository」をクリック

### 1-3. このフォルダをGitHubに送る
「コマンドプロンプト」を開いて、以下を順番にコピペして実行：

```
cd C:\Users\user\Documents\fasting-app
git remote add origin https://github.com/【あなたのGitHubユーザー名】/fasting-app.git
git add .
git commit -m "最初のアップロード"
git branch -M main
git push -u origin main
```

→ GitHubのユーザー名とパスワードを聞かれたら入力する

---

## STEP 2：Cloudflareに登録する

1. https://www.cloudflare.com を開く
2. 「Sign up」でアカウント作成（無料プランでOK）
3. メールアドレス確認まで完了させる

---

## STEP 3：データベースを作る

コマンドプロンプトで以下を実行：

### 3-1. Cloudflareにログイン
```
cd C:\Users\user\Documents\fasting-app
npx wrangler login
```
→ ブラウザが開くので「Allow」をクリック

### 3-2. データベースを作成
```
npm run db:create
```

→ 実行すると以下のような表示が出ます：
```
✅ Successfully created DB 'fasting-db'
{
  "uuid": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",  ← これをコピー！
  "name": "fasting-db"
}
```

### 3-3. wrangler.jsonc を更新する
`C:\Users\user\Documents\fasting-app\wrangler.jsonc` をメモ帳で開き、
`REPLACE_AFTER_CREATION` の部分を上でコピーした uuid に書き換える

例：
```
"database_id": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
```

### 3-4. テーブルを作成
```
npm run db:init
```

---

## STEP 4：アプリを公開する

### 4-1. Cloudflare Pagesに接続
1. https://dash.cloudflare.com を開く
2. 左メニューの「Workers & Pages」をクリック
3. 「Create」→「Pages」→「Connect to Git」をクリック
4. GitHubと連携して「fasting-app」リポジトリを選ぶ
5. 以下の設定にする：
   - **Framework preset**: Next.js
   - **Build command**: `npx @cloudflare/next-on-pages`
   - **Build output directory**: `.vercel/output/static`
6. 「Save and Deploy」をクリック

### 4-2. D1データベースを紐付ける
デプロイが終わったら：
1. Cloudflare Pages の「Settings」→「Bindings」を開く
2. 「Add」→「D1 database」をクリック
3. 以下を設定：
   - Variable name: `DB`
   - D1 database: `fasting-db`
4. 「Save」をクリック
5. 「Deployments」→「Retry deployment」で再デプロイ

---

## STEP 5：動作確認

1. Cloudflare Pages のダッシュボードにURLが表示される
   （例：`https://fasting-app-xxx.pages.dev`）
2. スマホでそのURLを開く
3. ホーム画面に追加する（iPhoneなら「共有」→「ホーム画面に追加」）

---

## 📱 使い方

### クライアントを追加する
1. 「👥 クライアント」タブ → 「＋ 追加」
2. 名前・ファスティング開始日・プランを入力
3. 「登録する」をクリック

### 今日送るメッセージを確認する
1. 「🏠 今日」タブを開く
2. 今日送るべきメッセージが自動で表示される
3. メッセージの「確認」→内容を確認してコピー
4. LINEに貼り付けて送信
5. チェックボックスをタップして送信済みにする

### テンプレートを見る
「📋 テンプレート」タブから全テンプレートをいつでも確認・コピーできます

---

## 🔄 テンプレートを修正したいとき

`C:\Users\user\Documents\fasting-app\lib\templates.ts` を開いて
`body:` の部分を書き換える → GitHubにプッシュすると自動で更新される

```
cd C:\Users\user\Documents\fasting-app
git add .
git commit -m "テンプレート修正"
git push
```

---

## ❓ 困ったときは

- アプリが開かない → Cloudflare Pages のダッシュボードでエラーを確認
- データが消えた → D1データベースは消えないので、URLを確認する
- テンプレートを変えたい → templates.ts を編集してgit push

何かあればClaudeに「fasting-appのここを直して」と相談してください！
